# quiz-session

## Purpose
퀴즈 진행 중 문제 표시, 답 입력, 이전/다음 네비게이션, 완료 처리 등 퀴즈 세션의 사용자 경험을 정의한다.

## Requirements

### Requirement: 문제 표시
시스템은 한 화면에 하나의 문제만 표시해야 한다(SHALL). 문제는 "A ○ B = ?" 형태로 표시하고, 사용자가 답을 입력할 수 있는 입력 필드를 제공해야 한다(SHALL). 숫자와 연산 기호는 다른 색상으로 구분하여 표시해야 한다(SHALL).

#### Scenario: 문제 화면 구성
- **WHEN** 퀴즈 화면에 진입할 때
- **THEN** 첫 번째 문제가 "A ○ B = ?" 형태로 표시되고 답 입력 필드가 활성화된다

#### Scenario: 숫자와 연산 기호 색상 구분
- **WHEN** 문제가 표시될 때
- **THEN** 숫자는 어두운 색(#2d3436), 연산 기호(+, −, ×, ÷, =, ?)는 하늘색(#5BC5F2)으로 표시된다

### Requirement: 진행 상황 표시
시스템은 현재 몇 번째 문제를 풀고 있는지 표시해야 한다(SHALL). 형식: "N / 20".

#### Scenario: 진행률 표시
- **WHEN** 사용자가 K번째 문제를 보고 있을 때
- **THEN** 화면에 "K / 20" 형태로 진행 상황이 표시된다

### Requirement: 다음 문제로 이동
시스템은 답을 입력하지 않아도 다음 버튼을 누르면 다음 문제로 이동할 수 있어야 한다(SHALL). 입력한 답은 저장되어야 한다(SHALL).

#### Scenario: 답 입력 후 다음 이동
- **WHEN** 사용자가 답을 입력하고 다음 버튼을 클릭할 때
- **THEN** 입력한 답이 저장되고 다음 문제가 표시된다

#### Scenario: 답 미입력 후 다음 이동
- **WHEN** 사용자가 답을 입력하지 않고 다음 버튼을 클릭할 때
- **THEN** 해당 문제는 미답으로 저장되고 다음 문제가 표시된다

### Requirement: 이전 문제로 이동
시스템은 이전 버튼을 누르면 이전 문제로 돌아갈 수 있어야 한다(SHALL). 이전에 입력한 답이 있으면 해당 답이 입력 필드에 복원되어야 한다(SHALL).

#### Scenario: 이전 문제로 돌아가기
- **WHEN** 사용자가 이전 버튼을 클릭할 때
- **THEN** 이전 문제가 표시되고, 기존에 입력한 답이 입력 필드에 복원된다

#### Scenario: 첫 번째 문제에서 이전 버튼
- **WHEN** 사용자가 첫 번째 문제에서 이전 버튼을 클릭할 때
- **THEN** 이전 버튼이 비활성화되어 있거나 아무 동작하지 않는다

### Requirement: 마지막 문제 완료 시 채점
시스템은 마지막 문제(20번)에서 다음/완료 버튼을 누르면 채점 결과 화면으로 이동해야 한다(SHALL).

#### Scenario: 모든 문제 완료 후 채점
- **WHEN** 사용자가 20번째 문제에서 완료 버튼을 클릭할 때
- **THEN** 모든 답이 채점되고 결과 화면으로 이동한다

### Requirement: Enter 키 지원
시스템은 답 입력 후 Enter 키를 누르면 다음 문제로 이동해야 한다(SHALL).

#### Scenario: Enter 키로 다음 문제 이동
- **WHEN** 사용자가 답 입력 필드에서 Enter 키를 누를 때
- **THEN** 다음 버튼을 클릭한 것과 동일하게 동작한다

### Requirement: 퀴즈 중 홈 이동
시스템은 퀴즈 진행 중 홈으로 바로 이동할 수 있는 버튼을 제공해야 한다(SHALL).

#### Scenario: 퀴즈 화면에서 홈으로 이동
- **WHEN** 사용자가 퀴즈 화면에서 홈으로 버튼을 클릭할 때
- **THEN** BGM이 중지되고 홈 화면으로 이동한다
