## ADDED Requirements

### Requirement: 더하기 문제 생성 (난이도 1~10)
시스템은 더하기 연산에 대해 난이도별로 다음 규칙에 따라 20문제를 랜덤 생성해야 한다(SHALL).

- 난이도 1: 한자리 + 한자리 (1~9 + 1~9)
- 난이도 2: 두자리 + 한자리 (올림 없음, 예: 23+4=27, 일의 자리 합 ≤ 9)
- 난이도 3: 한자리 + 두자리 (올림 없음, 예: 4+23=27, 일의 자리 합 ≤ 9)
- 난이도 4: 두자리 + 한자리 (올림 있음, 예: 27+5=32, 일의 자리 합 ≥ 10)
- 난이도 5: 한자리 + 두자리 (올림 있음, 예: 5+27=32, 일의 자리 합 ≥ 10)
- 난이도 6: 두자리 + 두자리 (일의 자리 합 ≤ 9, 십의 자리 합 ≤ 9)
- 난이도 7: 두자리 + 두자리 (일의 자리 합 ≥ 10, 십의 자리 합 ≤ 9)
- 난이도 8: 두자리 + 두자리 (각 자리 합 ≥ 10, 총합 ≤ 99)
- 난이도 9: 두자리 + 두자리 (합이 100~109)
- 난이도 10: 세자리 + 두자리 (100~999 + 10~99)

#### Scenario: 난이도 1 문제 생성
- **WHEN** 더하기 난이도 1로 문제를 생성할 때
- **THEN** 1~9 범위의 두 수의 덧셈 문제 20개가 생성된다

#### Scenario: 난이도 2 문제 생성 (올림 없음)
- **WHEN** 더하기 난이도 2로 문제를 생성할 때
- **THEN** 두자리수(10~99) + 한자리수(1~9)이며, 일의 자리 합이 9 이하인 문제 20개가 생성된다

#### Scenario: 난이도 6 문제 생성 (올림 없음)
- **WHEN** 더하기 난이도 6으로 문제를 생성할 때
- **THEN** 두자리수 + 두자리수이며, 일의 자리 합 ≤ 9이고 십의 자리 합 ≤ 9인 문제 20개가 생성된다

#### Scenario: 난이도 9 문제 생성 (합 100 초과)
- **WHEN** 더하기 난이도 9로 문제를 생성할 때
- **THEN** 두자리수 + 두자리수이며, 합이 100~109 범위인 문제 20개가 생성된다

#### Scenario: 난이도 10 문제 생성
- **WHEN** 더하기 난이도 10으로 문제를 생성할 때
- **THEN** 세자리수(100~999) + 두자리수(10~99)의 문제 20개가 생성된다

### Requirement: 빼기 문제 생성
시스템은 빼기 연산에 대해 더하기의 역연산으로 문제를 생성해야 한다(SHALL). 결과는 항상 0 이상이어야 한다(MUST). 난이도 구조는 더하기와 동일한 자릿수 조합을 따르되, 큰 수에서 작은 수를 빼는 형태로 구성한다.

#### Scenario: 빼기 문제의 결과가 음수가 아님
- **WHEN** 빼기 문제를 생성할 때
- **THEN** 모든 문제의 정답은 0 이상이어야 한다

#### Scenario: 빼기 난이도별 자릿수 대응
- **WHEN** 빼기 난이도 N으로 문제를 생성할 때
- **THEN** 더하기 난이도 N과 동일한 자릿수 조합의 뺄셈 문제 20개가 생성된다

### Requirement: 곱하기 문제 생성
시스템은 곱하기 연산에 대해 난이도별로 자릿수 조합을 조정하여 문제를 생성해야 한다(SHALL).

- 난이도 1: 한자리 × 한자리 (2~9 × 2~9)
- 난이도 2~3: 두자리 × 한자리 (결과가 작은 수)
- 난이도 4~5: 두자리 × 한자리 (결과가 큰 수)
- 난이도 6~7: 두자리 × 두자리 (결과가 작은 수)
- 난이도 8~9: 두자리 × 두자리 (결과가 큰 수)
- 난이도 10: 세자리 × 한자리

#### Scenario: 곱하기 난이도 1 문제 생성
- **WHEN** 곱하기 난이도 1로 문제를 생성할 때
- **THEN** 2~9 범위의 두 수의 곱셈 문제 20개가 생성된다

#### Scenario: 곱하기 난이도 10 문제 생성
- **WHEN** 곱하기 난이도 10으로 문제를 생성할 때
- **THEN** 세자리수(100~999) × 한자리수(2~9)의 문제 20개가 생성된다

### Requirement: 나누기 문제 생성
시스템은 나누기 연산에 대해 곱하기의 역연산으로 문제를 생성해야 한다(SHALL). 모든 나누기 문제는 나누어떨어져야 한다(MUST). 나누는 수는 0이 아니어야 한다(MUST).

#### Scenario: 나누기 문제가 나누어떨어짐
- **WHEN** 나누기 문제를 생성할 때
- **THEN** 모든 문제의 정답은 정수여야 한다

#### Scenario: 나누기 난이도별 문제 생성
- **WHEN** 나누기 난이도 N으로 문제를 생성할 때
- **THEN** 곱하기 난이도 N의 역연산에 해당하는 나눗셈 문제 20개가 생성된다

### Requirement: 문제 수
모든 연산과 난이도 조합에서 정확히 20문제를 생성해야 한다(SHALL).

#### Scenario: 20문제 생성
- **WHEN** 어떤 연산과 난이도 조합으로든 문제를 생성할 때
- **THEN** 정확히 20문제가 생성된다
