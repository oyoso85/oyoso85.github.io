## ADDED Requirements

### Requirement: 채점 결과 표시
시스템은 모든 문제를 완료한 후 맞춘 개수와 총 문제 수를 표시해야 한다(SHALL). 형식: "N / 20".

#### Scenario: 채점 결과 화면
- **WHEN** 퀴즈가 완료되고 결과 화면에 진입할 때
- **THEN** 맞춘 문제 수 / 총 문제 수(20)가 표시된다

#### Scenario: 미답 문제 처리
- **WHEN** 답을 입력하지 않은 문제가 있을 때
- **THEN** 미답 문제는 오답으로 처리된다

### Requirement: 점수 저장
시스템은 채점 완료 시 점수를 localStorage에 자동 저장해야 한다(SHALL). 저장 항목에는 연산 종류, 난이도, 맞춘 수, 총 문제 수, 날짜가 포함되어야 한다(SHALL).

#### Scenario: 점수 자동 저장
- **WHEN** 채점이 완료될 때
- **THEN** 점수가 `{ operation, level, score, total, date }` 형태로 localStorage에 저장된다

### Requirement: 최근 점수 20개 유지
시스템은 localStorage에 최대 20개의 점수만 유지해야 한다(SHALL). 20개를 초과하면 가장 오래된 점수를 삭제해야 한다(SHALL).

#### Scenario: 점수 개수 초과 시 오래된 점수 삭제
- **WHEN** 21번째 점수가 저장될 때
- **THEN** 가장 오래된 점수가 삭제되어 총 20개가 유지된다

### Requirement: 점수 이력 조회
시스템은 저장된 점수 이력을 조회할 수 있는 화면을 제공해야 한다(SHALL). 최근 순서로 표시하며, 연산 종류, 난이도, 점수, 날짜를 포함해야 한다(SHALL).

#### Scenario: 점수 이력 화면 표시
- **WHEN** 사용자가 점수 이력 메뉴를 클릭할 때
- **THEN** 최근 점수가 최신순으로 목록 표시되며, 각 항목에 연산 종류, 난이도, 점수(N/20), 날짜가 포함된다

#### Scenario: 점수 이력이 없을 때
- **WHEN** 저장된 점수가 없을 때
- **THEN** "아직 기록이 없습니다" 메시지가 표시된다

### Requirement: 결과 화면에서 네비게이션
시스템은 결과 화면에서 홈으로 돌아가기 버튼을 제공해야 한다(SHALL).

#### Scenario: 홈으로 돌아가기
- **WHEN** 사용자가 결과 화면에서 홈 버튼을 클릭할 때
- **THEN** 홈 화면으로 이동한다
