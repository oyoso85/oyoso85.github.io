## ADDED Requirements

### Requirement: TTS 문제 읽기
시스템은 문제가 표시될 때 TTS(Text-to-Speech)를 사용하여 문제를 음성으로 읽어주어야 한다(SHALL). TTS 읽기 속도는 기본 속도의 90%로 설정하여 이해도를 높여야 한다(SHALL). 음소거 상태에서는 TTS가 재생되지 않아야 한다(SHALL).

#### Scenario: 문제 전환 시 TTS 읽기
- **WHEN** 새로운 문제로 전환될 때 (다음/이전 버튼 클릭 또는 Enter 키)
- **THEN** 문제 텍스트 "A ○ B = ?"를 TTS로 읽는다 (예: "3 더하기 5는?")

#### Scenario: TTS 속도 조정
- **WHEN** TTS가 문제를 읽을 때
- **THEN** 기본 속도(rate=1.0)의 90% 속도(rate=0.9)로 재생된다

#### Scenario: 음소거 시 TTS 미재생
- **WHEN** 음소거 상태에서 새로운 문제로 전환될 때
- **THEN** TTS가 재생되지 않는다

## MODIFIED Requirements

### Requirement: 다음 문제로 이동
시스템은 답을 입력하지 않아도 다음 버튼을 누르면 다음 문제로 이동할 수 있어야 한다(SHALL). 입력한 답은 저장되어야 한다(SHALL). 정답/오답 효과음이 재생되는 경우, 효과음 재생이 완료된 후 다음 문제로 전환하고 TTS로 문제를 읽어야 한다(SHALL).

#### Scenario: 답 입력 후 다음 이동
- **WHEN** 사용자가 답을 입력하고 다음 버튼을 클릭할 때
- **THEN** 정답/오답 효과음이 재생 완료된 후 입력한 답이 저장되고 다음 문제가 표시되며 TTS로 읽힌다

#### Scenario: 답 미입력 후 다음 이동
- **WHEN** 사용자가 답을 입력하지 않고 다음 버튼을 클릭할 때
- **THEN** 효과음 없이 바로 다음 문제가 표시되고 TTS로 읽힌다

#### Scenario: 효과음과 TTS 순차 재생
- **WHEN** 사용자가 답을 입력하고 다음 버튼을 클릭할 때
- **THEN** 효과음이 먼저 재생되고, 완료 후 문제가 전환되며, 그 후 TTS가 재생된다 (효과음과 TTS가 동시 재생되지 않음)
