## MODIFIED Requirements

### Requirement: 진행 상황 표시
시스템은 현재 몇 번째 문제를 풀고 있는지 표시해야 한다(SHALL). 형식: "N / 20". 진행률은 화면 상단에 고정(fixed)되어 스크롤과 관계없이 항상 보여야 한다(SHALL). 음소거 버튼과 유사하게 화면 상단 왼쪽에 고정 배치해야 한다(SHALL).

#### Scenario: 진행률 표시
- **WHEN** 사용자가 K번째 문제를 보고 있을 때
- **THEN** 화면 상단 왼쪽에 고정된 "K / 20" 형태로 진행 상황이 표시된다

#### Scenario: 스크롤 시 진행률 유지
- **WHEN** 사용자가 화면을 스크롤할 때
- **THEN** 진행률 표시가 상단에 고정되어 항상 보인다

## ADDED Requirements

### Requirement: TTS 읽기 속도 조절
시스템은 TTS로 문제를 읽을 때 아동이 이해하기 쉬운 속도로 천천히 읽어야 한다(SHALL).

#### Scenario: 느린 속도로 문제 읽기
- **WHEN** TTS가 문제를 읽을 때
- **THEN** 기본 속도보다 느린 속도(rate 0.8)로 읽어준다

### Requirement: TTS 은/는 조사 구별
시스템은 TTS로 문제를 읽을 때 마지막 숫자의 받침 유무에 따라 '은?'/'는?'을 올바르게 구별해야 한다(SHALL).

#### Scenario: 받침 있는 숫자 뒤 조사
- **WHEN** 두 번째 숫자의 마지막 자릿수가 0, 1, 3, 6, 7, 8일 때
- **THEN** "은?" 조사를 사용한다 (예: "3 더하기 1 은?")

#### Scenario: 받침 없는 숫자 뒤 조사
- **WHEN** 두 번째 숫자의 마지막 자릿수가 2, 4, 5, 9일 때
- **THEN** "는?" 조사를 사용한다 (예: "3 더하기 2 는?")
